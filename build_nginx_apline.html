Nginx 静态编译与 Supervisord 成功经验

编译环境

  • 基础环境：Ubuntu 22.04 (GitHub Actions Runner)
  • 依赖：build-essential, wget, tar, zlib1g-dev
  • 静态库：OpenSSL 3.0.12, PCRE2 10.42

关键编译步骤

# 编译 OpenSSL 静态库
./config no-shared && make -j$(nproc)

# 编译 PCRE2 静态库
./configure --disable-shared --enable-static && make -j$(nproc)

# 编译 Nginx 静态二进制
./configure --prefix=/opt/nginx \
  --with-http_ssl_module \
  --with-http_v2_module \
  --with-http_gzip_static_module \
  --with-pcre=../pcre2-src \
  --with-openssl=../openssl-src \
  --with-openssl-opt=no-shared \
  --with-cc-opt="-static -static-libgcc" \
  --with-ld-opt="-static"
make -j$(nproc)

验证结果

ldd /opt/nginx/sbin/nginx
/lib/ld-musl-x86_64.so.1: /opt/nginx/sbin/nginx: Not a valid dynamic program

说明已完全静态编译，无需依赖动态库。

运行问题与解决

  • 警告：user appuser; 指令在非 root 下无效 → 已删除。
  • 错误：缺少 mime.types 文件 → 已补齐到 /opt/nginx/conf/mime.types。
  • 日志目录权限 → 使用 chown -R appuser:appgroup /opt/nginx 修复。

经验总结

  1. 必须静态编译 OpenSSL 与 PCRE2，避免 Alpine 缺库问题。
  2. 去掉 user 指令，避免非 root 下的警告。
  3. 确保 mime.types 文件存在，否则 nginx 无法启动。
  4. 日志与 PID 文件目录必须赋权给运行用户。
  5. supervisord 配置中使用 daemon off; 保证前台运行。
